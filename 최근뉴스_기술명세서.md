# 최근 뉴스 기능 기술 명세서

## 1. 개요

| 항목 | 내용 |
|------|------|
| 기능명 | 최신 야구 뉴스 조회 및 음성 읽기 |
| 목적 | 프로야구 관련 최신 뉴스 5건을 자동으로 수집해 목록으로 보여주고, 사용자가 원할 때 음성으로 읽어준다 |
| 대상 화면 | 알람 앱 우측 패널 (`NewsPanel`) |
| 갱신 방식 | 실시간 호출 (캐싱 없음) — 페이지 진입 시 및 새로고침 클릭 시마다 API 재호출 |
| 인증/사용자 구분 | 없음 (개인용 단일 사용자 기준) |

## 2. 전체 아키텍처

```
[Browser: React Component]
        │
        │ ① GET /api/news
        ▼
[Next.js Route Handler (서버)]
        │
        │ ② GET openapi.naver.com/v1/search/news.json
        │    (X-Naver-Client-Id / Secret 헤더)
        ▼
[네이버 검색 API]
        │
        │ ③ JSON 응답 (뉴스 원본)
        ▼
[Route Handler: HTML 태그/엔티티 정제 → JSON 재구성]
        │
        │ ④ { items: [...] }
        ▼
[Browser: 상태 저장 → 목록 렌더링]
        │
        │ ⑤ 사용자가 ▶ 클릭
        ▼
[Web Speech API (SpeechSynthesis)] → 브라우저가 직접 음성 합성 (서버 경유 없음)
```

핵심 설계 원칙은 **API 키를 브라우저에 절대 노출하지 않는 것**이다. 네이버 API는 CORS를 허용하지 않고 키 노출 위험도 있으므로, 프론트엔드가 네이버 API를 직접 호출하지 않고 반드시 Next.js 서버 라우트(`/api/news`)를 경유한다.

## 3. 데이터 흐름 상세

### 3.1 요청 (서버 → 네이버)

- **엔드포인트**: `https://openapi.naver.com/v1/search/news.json`
- **쿼리 파라미터**
  - `query`: `프로야구` (URL 인코딩)
  - `display`: `5` (고정, 요구사항: 5건)
  - `sort`: `date` (최신순)
- **요청 헤더**
  - `X-Naver-Client-Id`: 서버 환경변수 `NAVER_CLIENT_ID`
  - `X-Naver-Client-Secret`: 서버 환경변수 `NAVER_CLIENT_SECRET`
- **캐시 정책**: `cache: 'no-store'` — 매 요청마다 실시간 조회

### 3.2 응답 정제 (서버 내부)

네이버 API 원본 응답 필드:

```json
{
  "items": [
    {
      "title": "<b>KBO</b> 리그 소식...",
      "originallink": "https://...",
      "link": "https://news.naver.com/...",
      "description": "오늘 경기에서 &quot;선수&quot;가...",
      "pubDate": "Fri, 24 Jul 2026 09:00:00 +0900"
    }
  ]
}
```

정제 규칙:

| 원본 패턴 | 처리 |
|-----------|------|
| `<b>`, `</b>` | 제거 (검색어 강조 태그) |
| `&quot;` | `"` 로 치환 |
| `&amp;` | `&` 로 치환 |
| `&lt;`, `&gt;` | `<`, `>` 로 치환 |
| `&nbsp;` | 공백으로 치환 |
| `&#39;` | `'` 로 치환 |
| 링크 | `originallink` 우선 사용, 없으면 `link` |

### 3.3 클라이언트 응답 스키마

`GET /api/news` 최종 응답:

```json
{
  "items": [
    {
      "title": "string",
      "description": "string",
      "link": "string (URL)",
      "pubDate": "string (RFC 1123)"
    }
  ]
}
```

오류 시:

```json
{ "error": "string" }
```

| 상황 | HTTP 상태 |
|------|-----------|
| 정상 | 200 |
| 서버 환경변수 미설정 | 500 |
| 네이버 API 오류 응답 | 네이버 응답 상태 그대로 전달 |
| 네트워크/예외 | 500 |

## 4. 프론트엔드 컴포넌트 명세 (`NewsPanel.tsx`)

### 4.1 상태

| 상태 | 타입 | 설명 |
|------|------|------|
| `items` | `NewsItem[]` | 현재 표시 중인 뉴스 목록 (최대 5건) |
| `loading` | `boolean` | 조회 중 여부 |
| `error` | `string \| null` | 오류 메시지 |
| `playingIndex` | `number \| null` | 현재 읽는 중인 항목 인덱스. `-1`은 전체 읽기 중 |

### 4.2 동작

1. **최초 마운트**: `useEffect`에서 `fetchNews()` 자동 호출
2. **새로고침 버튼**: `fetchNews()` 재호출, 로딩 중에는 버튼 비활성화 + "불러오는 중…" 표시
3. **개별 읽기(▶)**: 클릭 시 해당 항목의 `title + description`을 `speak()`에 전달. 같은 항목을 재클릭하면 정지
4. **전체 듣기**: 5건을 "N번째 뉴스. 제목. 설명." 형식으로 이어붙여 한 번에 읽음. 재생 중 클릭 시 정지
5. **빈 상태 / 오류**: 목록이 비어 있으면 안내 문구, 오류 발생 시 오류 메시지 표시

## 5. 음성 읽기 명세 (`useSpeech.ts`)

| 항목 | 값 |
|------|-----|
| API | Web Speech API `SpeechSynthesisUtterance` |
| 언어 | `ko-KR` 고정 |
| 속도(`rate`) | `1` (기본값) |
| 서버 경유 여부 | 없음 (브라우저가 로컬에서 직접 처리) |
| 동시 재생 제어 | 새 재생 시작 시 `speechSynthesis.cancel()`로 이전 재생 취소 후 시작 |
| 지원 여부 확인 | `'speechSynthesis' in window` 체크 → 미지원 브라우저는 버튼 비활성화 + 안내 문구 |

**제약사항**: 음성 품질과 목소리는 사용자의 OS/브라우저 내장 TTS 엔진에 의존하며 기기마다 다르게 들릴 수 있다. 더 일관된 품질이 필요할 경우 네이버 Clova Voice 또는 Google Cloud TTS로 교체 가능하며, 이 경우 서버 프록시 라우트(`/api/tts` 등) 추가가 필요하다.

## 6. 환경변수

| 변수명 | 위치 | 노출 범위 | 설명 |
|--------|------|-----------|------|
| `NAVER_CLIENT_ID` | 서버 (`.env.local` / Vercel) | 서버 전용 | 네이버 오픈API 클라이언트 ID |
| `NAVER_CLIENT_SECRET` | 서버 (`.env.local` / Vercel) | 서버 전용 | 네이버 오픈API 시크릿 |

`NEXT_PUBLIC_` 접두사를 절대 붙이지 않는다. 붙일 경우 브라우저 번들에 포함되어 키가 노출된다.

## 7. 보안 고려사항

- 네이버 API 키는 서버 환경변수로만 관리하며, Route Handler 내부에서만 읽는다.
- 프론트엔드는 `/api/news`만 호출하고 외부 API 주소나 키를 알 수 없다.
- 응답 정제 단계에서 HTML 태그를 제거하므로, 목록 렌더링 시 별도의 XSS 위험은 낮다 (React가 텍스트를 기본적으로 이스케이프 처리).
- 외부 링크(`item.link`)는 `target="_blank" rel="noopener noreferrer"`로 열어 `window.opener` 공격을 방지한다.

## 8. 한계 및 향후 확장

| 항목 | 현재 | 확장 방향 |
|------|------|-----------|
| 캐싱 | 없음 (매번 실시간 호출) | Supabase 테이블 + `pg_cron`으로 주기 갱신 후 캐시 조회 |
| 검색어 | `프로야구` 고정 | 팀명/선수명 등 사용자 지정 검색어 입력 UI 추가 |
| 읽어주기 품질 | 브라우저 내장 TTS | Clova Voice / Google Cloud TTS 연동 (서버 프록시 필요) |
| 뉴스 개수 | 5건 고정 (`display=5`) | 쿼리 파라미터로 조절 가능하도록 UI 추가 |
| 오류 재시도 | 없음 (수동 새로고침만) | 지수 백오프 자동 재시도 로직 추가 |

## 9. 관련 파일

| 파일 | 역할 |
|------|------|
| `app/api/news/route.ts` | 네이버 API 호출 및 응답 정제 (서버) |
| `components/NewsPanel.tsx` | 뉴스 목록 UI 및 읽기 제어 (클라이언트) |
| `hooks/useSpeech.ts` | Web Speech API 래핑 훅 |
